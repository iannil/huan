## 关键技术决策

### 定位拆段（ADR 0010）

v0.x 实际范围 = **local-first single-user content engine with built-in admin**。"Path toward all-CMS replacement"（图片管线 / 多用户 / 资源管道）列为 v1.x+ roadmap，**非 v1.0 范围**。这撤回了 2026-06-26 把定位升级为"替代所有 CMS"的过度宣称。

### Admin 安全三层（ADR 0011）

- **L1 fail-fast**：`huan serve --bind 0.0.0.0` 无 `HUAN_ADMIN_TOKEN` 拒启。强制用户显式配 token 才能暴露 admin。
- **L2 token 中间件**：所有 `/admin/api/*` 请求必须携带 token（loopback 缺失 env 时自动生成 + stderr 打印）。`crypto/subtle.ConstantTimeCompare` 防时序攻击。
- **L4 audit log**：所有写操作（content/settings/media 共 7 种）append 到 `memory/daily/{date}.md`，记录 before→after SHA256。
- **否决 L3 CSRF cookie**：loopback 价值小，推 v1.x 多用户场景。

实施中发现的**关键 bug**：初版 `ResolveToken` 一步到位（env→auto-gen），导致 non-loopback + 无 env 时 auto-gen 错误地满足 L1 检查。修复为两步（env-only read → CheckBindSafety → 然后才 auto-gen），保证 fail-fast 不被绕过。

### 模板函数三档策略（gate 2）

实证：zhurongshuo layouts/themes **零调用**这些 broken funcs。所以策略为：
- **真实现 7 个**（cheap + 可能被未来模板作者用到）：markdownify/now/highlight/dateFormat/relURL + absLangURL/relLangURL（多语 deferred to v1.x）
- **Panic-on-call 4 个**（expensive + 当前无需求）：emojify/pluralize/singularize/apply
- **守护测试 TestNoSilentNoOpFuncs**：FuncMap 每个 entry 必须在 testedFuncs 或 panicOnCallFuncs 之一，挡住未来回归

### BuildSite 纯抽取（gate 5）

否决 Pipeline 框架（Stage interface + Run）—— over-engineering，technical-plan §4.11 build-time plugin 明示 YAGNI。采用纯抽取：pipeline struct 持有 7 个跨 stage 共享值（cfg/site/siteCtx/lookup/writer/renderer/result），避免 ~10 arg 函数签名。
