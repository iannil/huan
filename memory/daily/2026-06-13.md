### stage 2 phase 3c 完成：tags/index.xml + posts/index.xml 字节对齐（22:50）

**修改文件**：
- `/Users/rong.zhu/Code/huan/internal/build/context.go`：重写 `BuildTaxonomyContext`，让 `/tags/` RSS 的 `RegularPages` 输出 term stub pages（Hugo 行为），而非 site.RegularPages。
- `/Users/rong.zhu/Code/huan/internal/content/tree.go`：auto-create section 时用 `cases.Title(language.English)` 处理目录名（"posts" → "Posts"，"my-notes" → "My Notes"）。
- 新增 `internal/build/context_test.go`：4 个测试覆盖 term stub、URL 百分比编码、permalink、按 date desc + collator 排序。
- 新增 `internal/content/tree_test.go::TestBuildTree_AutoCreatedSectionTitle*`：两个测试覆盖单字 / 多字 section。

**根因总结**：
1. **tags/index.xml**：Hugo 的 taxonomy-list RSS 迭代 `.Pages`（即 `.RegularPages`），对 taxonomy list 而言这些 page 是 **term pages**（每个 tag 一个 stub），而非站点普通文章。huan 错误地把 `siteCtx.RegularPages`（普通文章列表）赋给了 taxonomy context。
2. **posts/index.xml**：当 section 缺 `_index.md` 时，huan auto-creates section page 用了目录原名（"posts"），而 Hugo 把目录名 title-case 化（"Posts"）。

**关键决策**：
- **Term stub 排序**：Hugo 的 `DefaultPageSort` = Weight → Date desc → LinkTitle(collator asc) → Path asc。huan 用 `i18n.BuildCollator(cfg.LanguageCode)` 复现 zh-cn collator 的 tiebreak，验证 共识 < 观点、曲率 < 权力（CJK 字符按 collator，不是 UTF-8 byte 也不是全拼音比较）。
- **CJK URL 编码**：Hugo 在 permalink 中对 CJK 做 percent-encoding（共识 → `%E5%85%B1%E8%AF%86`），文件系统路径保留 CJK。huan 用 `url.PathEscape(term)` 给 stub permalink。
- **双斜杠避免**：`siteCtx.BaseURL` 已带尾 `/`，所以用 `BaseURL + "tags/"` 而非 `BaseURL + "/tags/"`。
- **Channel permalink**：补上 `Permalink` / `RelPermalink`，让 RSS channel `<link>` 不再为空。

**4-mode diff-build**：
- Identical files: 972 → **974**（+2：tags/index.xml、posts/index.xml 字节对齐）
- Files with real diffs: 997 → **995**

**未解决（不在 phase 3c 范围）**：
- 单个 tag 的 RSS（`tags/{cjk}/index.xml`）仍然 link/guid 用 raw CJK；这是更广的 "CJK URL encoding" 议题，影响所有 term RSS 文件（不只 tags/index.xml），留待后续阶段统一处理。
- `BuildEmptyTaxonomyContext`（`/categories/`）仍未修相同的 RegularPages 内容定义差（如果将来要 byte-match categories/index.xml，需要同样改造）。

### 经验教训（phase 3c）

1. **Hugo 的 RSS 用 .RegularPages，taxonomy list 上 .RegularPages 是 term stubs**：这是个反直觉点。taxonomy list page 的 .Pages / .RegularPages 不是文章列表，而是 term 列表（每个 term 一个 stub）。Hugo 的 DefaultPageSort 在 stub 上 = 按 term 的最新 page Date desc + LinkTitle collator。
2. **section title 自动生成要 title-case**：Hugo 用 `cases.Title(language.English)` + 把 `-`/`_` 换空格。简单 `strings.Title` 已 deprecated，必须用 `golang.org/x/text/cases`。
3. **collator.CompareString 而非 strings.Compare**：CJK tiebreak 必须用 collator（已建立的 i18n.BuildCollator），否则顺序错。
