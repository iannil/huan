### 经验教训

1. **Port 上游算法必须查 collator / locale / sort 关键词**：Go 标准 `strings.Compare` 是 byte-level，CJK 字符按 UTF-8 编码排序，与 Hugo 的 collate 拼音序完全不同。
2. **排序修复要覆盖所有调用路径**：phase 2 初版只修了 BuildTree 内的 sortPagesDefault，遗漏了 taxonomy.BuildAll 调用点（build.go:173），导致 154 个 tags RSS 文件未对齐。**含义**：grep 修复函数名时必须检查所有数据流路径，不能假设"修了源头就全对"。
3. **实证失败回头调查而非继续 Phase 4**：phase 1 教训复用——phase 3 验证发现 tags RSS 仍不同时立刻 BLOCKED + 调查根因，避免错误基础上继续文档化。
