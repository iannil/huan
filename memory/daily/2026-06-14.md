## grill-me: i18n 翻译插件设计（10 个决策收敛）

用户提问：`规划和设计一个新插件。zhurongshuo 的默认使用语言是中文，但是需要在 deploy 时，支持将中文翻译为英文。根据网站访问者使用的浏览器语言，自动显示中文版或英文版。`

### 10 个决策（按 grill-me 顺序）

| # | 决策点 | 收敛 |
|---|---|---|
| Q1 | 语言路由实现 | **A：Worker 边缘检测**（Cloudflare Worker 拦截根路径，读 `Accept-Language`，302 跳转） |
| Q2 | 英文版 URL 形状 | **A：子路径 `/en/`**（zhurongshuo.com/posts/foo/ vs /en/posts/foo/） |
| Q3 | Worker 检测机制 | **A：302 + Cookie 记忆**（首次访问检测 + set cookie，回访 cookie 主导） |
| Q4 | Worker 默认配置 | cookie name=`lang` / value=`zh`/`en` / domain=`.zhurongshuo.com` / 1 年 / SameSite=Lax / HttpOnly；Accept-Language first-preference wins；无 geo fallback；`/en/*` 与 SEO 资源不拦截 |
| Q5 | 翻译引擎首发 | **反转推荐**：从原推荐 Anthropic Claude Sonnet 4.6 → 改为 **本地 Qwen3-Next-80B-A3B-Instruct（q4_K_M）** 首发；保留 Claude 作为质量补强通道。理由：M5 Max + 128GB 硬件实测 80B-A3B 反超 14B 速度（MoE 稀疏激活），无 thinking 污染，翻译更精炼 |
| Q6 | 翻译产物存放 + 增量 | **A：Sidecar `.en.md`**（Hugo 既有约定）+ frontmatter `source_hash: sha256(原文 body)` + `model` + `translated_at`；per-file 失败不阻断整体 |
| Q7 | 翻译内容范围 | 21 项资产策略：post body/title/description 走 LLM；summary 从英文 body 重抽取；**tag 走手工字典 `i18n/terms.yaml` + fail-fast** + `huan translate terms --propose`；menu/UI 走已有 i18n bundle；site title "祝融说" 与 author "祝融" **不译**；subtitle/description/keywords 走 LLM 一次性缓存到 plugin config；hreflang + per-lang RSS + 统一 sitemap |
| Q8 | Prompt + 质量守门 | system prompt 文件化（`i18n/translate-prompt-zh-en.md`）；XML 标签 `<title>...</title><body>...</body>` 单次调用；GLOSSARY 预注入 + 后验证；硬通过三项（XML 解析 + 语言检测 + Markdown 结构计数）；风格可配置（`system_prompt_file`）|
| Q9 | Build pipeline 集成 | **i18n build 是 core，translate 是 plugin**：`internal/build/i18n.go` 是 huan core 特性（与 RSS/sitemap 同级）；`internal/translate/qwen3/` 是 plugin（首期 Translator capability）。huan.yaml 加 `languages:` 块。MultiSite 构建并行 site tree。内链 rewriting（en 版本加 `/en` 前缀）。hreflang 模板 helper + 兜底注入 |
| Q10 | 风险 + stale + 验收 + observability | stale 检测 strict（CI）+ 柔性（本地）；抽样 75 篇 ≥ 95% 通过率；回滚矩阵（i18n Worker 永不阻断主 deploy）；复用 `internal/observability/` 包；v1 验收 6 维度（功能/质量/SEO/回归/文档/observability） |

### 关键架构 fork（与历史 grill-me 一致风格记录）

1. **i18n build = core，translate = plugin**：与 Hugo 模型对齐；plugin 只产 `.en.md`，core 消费；i18n 多语言是基础设施（与 RSS/sitemap/taxonomy 同级）
2. **本地模型优先（Qwen3-Next-80B）而非 API（Claude）**：基于硬件实证（M5 Max + 128GB），MoE 架构在 Apple Silicon 上是甜点；零边际成本 + 数据不外流 + 翻译产物可重现
3. **Tag 手工字典而非 LLM 自由翻译**：tag 是站点架构（影响 URL / taxonomy / RSS），术语一致性必须强约束；LLM 仅 propose，user approve
4. **Sidecar `.en.md` 而非并行目录树**：Hugo 既有约定；文件成对存放便于 review；删除/重译独立
5. **翻译产物入 git**：CI 无 GPU，强制工作流"本地翻译 → commit → push → CI 构建"；翻译产物是"已发布内容"非"构建产物"
