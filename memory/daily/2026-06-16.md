## 盘古之白（CJK↔西文自动间距）处理

经 grill-me 收敛，决策树如下：

- **层次**：表现层（CSS），不改 HTML 文本 —— 避免破坏 ADR 0001 三维度等价基线。
- **机制**：原生 `text-autospace`（CSS Text Module L4），不引 JS（pangu.js 会往 DOM 插文本节点 = 变相内容层 + FOUC）。老浏览器不支持则优雅退化为「无间距」。
- **范围**：全局 `body { text-autospace: normal; }`；显式排除 `code, pre, kbd, samp { text-autospace: no-autospace; }`（保等宽对齐）。对 en 页面是天然 no-op（纯西文无 CJK 边界）。
- **不做**：`text-spacing-trim`（标点挤压）—— 另一个独立排版决策，影响每篇旧文标点观感，本次不搭车。
- **落地**：改源 `zhurongshuo/static/css/zozo.css`（现有 `html[lang="en"]` 差异化块之后，~第 60 行）。`docs/css/` 下副本是构建产物，重建覆盖，不手改。
- **等价影响**：无。CSS 在 static，Hugo 与 huan 都原样拷贝同一份 → 两边输出 CSS 字节一致，三维度等价不受影响；非单边 better-than-Hugo，是主题层一视同仁改动，无需 ADR。

实现：`zozo.css` 追加两条规则（body 开启 + 代码区关闭），取标准值 `normal`，不加旧草案 `ideograph-alpha` 兜底（保持干净）。
