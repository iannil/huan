## 新增：英文站按栏目隐藏（excludeSections）

**需求**：英文站不显示未翻译/不翻译栏目（books/practices/gallery/hidden）的入口与页面；中文站不变。用户选「按清单隐藏」+「huan.yaml 语言配置」机制（即使 books/practices 已有部分 .en.md 也整体隐藏）。

**根因**：`BuildMultiSite` 的列表页/sitemap 由「过滤后页面集」自动生成，0 英文页栏目(gallery/hidden)已天然不在 en；但 books/practices 有 en 页所以仍渲染，且入口（nav 菜单 / header social 图标）来自全局 config 无语言感知，URL 的 /en/ 前缀是渲染后 canonify 加的（模板函数看到原始 `/books/`）。

**实现**（配置驱动，声明式）：
- `internal/config/languages.go`：`LanguageConfig` 增 `ExcludeSections []string`；新增纯函数 `TopSection(url)` + 方法 `IsSectionExcludedForLang(code,url)`（先剥语言 BaseURL 前缀再取顶层段）。
- `internal/build/multisite.go`：非默认语言 PageFilter 追加「顶层 section ∈ excluded 则剔除」→ 内容页/列表页/sitemap 一并消失。
- `internal/template/funcs.go`：新增 `sectionExcluded $ url` 模板函数（读 `ctx.Site.Config`+`LanguageCode`，nil 安全降级），注册 FuncMap。
- zhurongshuo 主题：`layouts/partials/nav.html`、`header.html` 用 `{{ if not (sectionExcluded $ .URL) }}` 包裹入口；`huan.yaml` 的 `languages.en` 加 `excludeSections: [books, practices, gallery, hidden]`。

**端到端验证**：en 的 books/practices/gallery/hidden → 404、social 仅剩 products+about、nav 无 books/practices、en sitemap 0 条排除栏目；中文站 social/栏目/页面全不变。`go build/test ./...` 全绿（新增 config/build/template 单测）。

**边界**：已译 en 正文里指向这些栏目的站内链接不自动隐藏（入口已覆盖 nav/social/列表页主要场景）。

## 修复：`huan serve` 不区分中英文（多语言）

**问题**：`huan serve` 在多语言配置（`huan.yaml` 含 `languages:` 块）下只构建默认语言（zh-cn），不产出英文版，`/en/` 404。

**根因**：i18n 多语言系统（commit `4d385e2 feat(i18n): multilingual translation system v1`）是在所有 serve 相关提交之后才加入的。`huan build`（`cmd/huan/main.go:86`）会判断 `cfg.IsMultiLanguage()` 分发到 `BuildMultiSite`，但 `serve.go` 从未更新，初次构建与热重建都直接调 `BuildSite`（单语言路径）。

**修改**：
- `internal/build/multisite.go`：`BuildMultiSite` 现在在 master 层应用 `opts.BaseURLOverride`（serve 传入的 dev URL），再在 per-language `langOpts.BaseURLOverride = ""` 清空，避免 `BuildSite`（`build.go:101`）覆盖逐语言 `/en/` 前缀。这是 serve dev URL 与多语言 baseURL 前缀的耦合点。
- `cmd/huan/serve.go`：开头 `config.Load` 一次，引入 `runBuild` 闭包按 `IsMultiLanguage()` 分发到 `BuildMultiSite`/`BuildSite`，替换初次构建与 `doRebuild` 两处调用。新增 `internal/config` import。
- `internal/build/multisite_test.go`：新增 `TestBuildMultiSite_BaseURLOverridePerLanguage`，用 `sitemap.xml`（模板无关、内嵌 baseURL）断言默认语言 → `<override>/`、en → `<override>/en/`。

**测试经验**：build 全量渲染依赖 layouts 模板，内置 9 个模板缺 `single.html`，content 页渲染为 0；`sitemap.xml` 由内置模板渲染且内嵌 baseURL，是多语言构建里稳定、模板无关的断言目标。

**验证**：`go build ./...` + `go test ./internal/build/... ./cmd/huan/...` 全绿。端到端需在含 `languages:` 块且有 `.en.md` sidecar 的源目录上手动 `huan serve` 验证 `/` 与 `/en/`。
